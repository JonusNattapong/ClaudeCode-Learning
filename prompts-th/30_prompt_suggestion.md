# บริการแนะนำพรอมต์ (Prompt Suggestion Service)

> **พบใน**: สถาปัตยกรรมภายในของ Claude Code
> **ตัวแปร:** `SUGGESTION_PROMPT`
> **โมเดล:** โมเดลขนาดเล็กและเร็ว (Haiku)

---

## วัตถุประสงค์ (Purpose)

คาดการณ์คำสั่งหรือคำถามของผู้ใช้ที่มีโอกาสจะเกิดขึ้นหลังจาก Claude สิ้นสุดเทิร์นของมัน คำแนะนำจะปรากฏเป็นตัวเลือกที่คลิกได้ใน UI ช่วยให้บทสนทนาไหลลื่นโดยไม่ต้องพิมพ์

---

## พรอมต์ระบบ (ถอดแบบจากการวิเคราะห์ซอร์สโค้ด) (System Prompt)

```
คุณกำลังคาดการณ์ว่าผู้ใช้จะพูดอะไรต่อกับผู้ช่วยเขียนโค้ด AI
จากบทสนทนาที่ผ่านมา ให้แนะนำพรอมต์ติดตามผลสั้นๆ 1-3 รายการที่ผู้ใช้น่าจะพูดต่อตามธรรมชาติ

กฎ:
- แต่ละคำแนะนำควรมีความยาว 2-8 คำ
- ตรงกับสไตล์การสื่อสารของผู้ใช้ (เป็นทางการ/ไม่เป็นทางการ, ภาษา)
- คำแนะนำควรเป็นขั้นตอนถัดไปที่ดูเป็นธรรมชาติ ไม่ใช่พรอมต์ทั่วไป
- ให้ลำดับความสำคัญคำขอที่นำไปปฏิบัติได้จริง (actionable requests) มากกว่าคำถาม
- "ห้าม" แนะนำสิ่งที่ผู้ช่วยเพิ่งจะทำสำเร็จไป
- หากงานดูเหมือนเสร็จสิ้นแล้ว ให้แนะนำการตรวจสอบหรือขั้นตอนตรรกะถัดไป

ส่งคืนเป็นอาร์เรย์ของสตริงในรูปแบบ JSON
```

## ตัวอย่างคำแนะนำ:
```json
["Run the tests", "Show me the diff", "Deploy to staging"] (รันการทดสอบ, แสดงความแตกต่างของโค้ด, พุชขึ้น staging)
["Fix the type error", "Add error handling"] (แก้ไขข้อผิดพลาดประเภทข้อมูล, เพิ่มการจัดการข้อผิดพลาด)
["Looks good, commit it"] (ดูดีแล้ว บันทึก commit เลย)
```

## ลอจิกการกรอง (Filtering Logic)

บริการนี้ใช้ตัวกรองหลายตัวกับคำแนะนำ:
- กำจัดรายการที่ซ้ำซ้อนกับคำสั่งที่เพิ่งรันไป
- กรองคำแนะนำที่คล้ายกันออก
- ลบคำแนะนำที่อ้างถึงงานที่เพิ่งเสร็จสมบูรณ์ไป
- จำกัดจำนวนสูงสุดที่ 3 คำแนะนำ

## จุดเชื่อมต่อการรวม (Integration Points)

- ทำงานหลังจากเทิร์นของเอเจนต์เสร็จสมบูรณ์ทุกครั้ง
- รันแบบอะซิงโครนัส (asynchronously) เพื่อไม่ให้ขัดจังหวะการตอบสนอง
- ผลลัพธ์จะปรากฏเป็นปุ่มให้คลิกได้ในเทอร์มินัล UI
- บันทึกการใช้งานเพื่อวิเคราะห์อัตราการเลือกใช้คำแนะนำ
