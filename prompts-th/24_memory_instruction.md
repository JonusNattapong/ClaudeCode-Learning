# คำแนะนำความจำ (ระบบ CLAUDE.md) (Memory Instruction)

> **พบใน**: สถาปัตยกรรมภายในของ Claude Code
> **ตัวแปร:** `MEMORY_INSTRUCTION_PROMPT`

---

## วัตถุประสงค์ (Purpose)

คำแนะนำระดับแมโคร (meta-instruction) ที่ห่อหุ้มไฟล์ความจำ CLAUDE.md ทั้งหมดที่ถูกโหลดเข้าไปในพรอมต์ระบบ บรรทัดเดียวเพียงพอที่จะระบุว่าคำแนะนำที่ผู้ใช้จัดเตรียมไว้ให้นั้นมีความสำคัญเหนือกว่าพฤติกรรมเริ่มต้นของระบบ

---

## พรอมต์ (Prompt)

```
คำแนะนำสำหรับ codebase และคำแนะนำจากผู้ใช้แสดงอยู่ด้านล่าง โปรดปฏิบัติตามคำแนะนำเหล่านี้อย่างเคร่งครัด
สำคัญมาก: คำแนะนำเหล่านี้จะอยู่เหนือกว่า (OVERRIDE) พฤติกรรมเริ่มต้นใดๆ และคุณจะต้องปฏิบัติตามพวกมันอย่างแม่นยำตามที่เขียนไว้
```

---

## ลำดับการโหลดไฟล์ความจำ (Memory File Loading Order)

ไฟล์จะถูกโหลดตามลำดับความสำคัญย้อนกลับ (อันล่าสุด = ความสำคัญสูงสุด):

1. **ความจำที่จัดการ (Managed memory)** (`/etc/claude-code/CLAUDE.md`) — คำแนะนำระดับโลกสำหรับผู้ใช้ทุกคน
2. **ความจำของผู้ใช้ (User memory)** (`~/.claude/CLAUDE.md`) — คำแนะนำส่วนตัวระดับโลกสำหรับทุกโปรเจกต์
3. **ความจำโปรเจกต์ (Project memory)** (`CLAUDE.md`, `.claude/CLAUDE.md`, `.claude/rules/*.md` ในรูทของโปรเจกต์) — ถูกตรวจสอบเข้าสู่ codebase
4. **ความจำภายใน (Local memory)** (`CLAUDE.local.md` ในรูทของโปรเจกต์) — คำแนะนำส่วนตัวเฉพาะโปรเจกต์

---

## การค้นหาไฟล์ (File Discovery)

- ความจำของผู้ใช้จะถูกโหลดจาก `~/.claude/`
- ไฟล์โปรเจกต์และไฟล์ภายในจะถูกค้นหาโดยการสำรวจจากไดเรกทอรีปัจจุบันย้อนขึ้นไปยังระดับรูท
- ไฟล์ที่อยู่ใกล้กับไดเรกทอรีปัจจุบันมากกว่าจะมีความสำคัญสูงกว่า (ถูกโหลดทีหลัง)
- `CLAUDE.md`, `.claude/CLAUDE.md` และไฟล์ `.md` ทั้งหมดใน `.claude/rules/` จะถูกตรวจสอบในแต่ละไดเรกทอรี

## คำสั่ง @include (@include Directive)

ไฟล์ความจำเป็นกึ่งรองรับการรวมไฟล์แบบสลับสาย (transitive file inclusion):

- ไวยากรณ์ (Syntax): `@path`, `@./relative/path`, `@~/home/path`, หรือ `@/absolute/path`
- ทำงานเฉพาะในโหนดข้อความปลายทาง (leaf text nodes) เท่านั้น (ไม่ได้ทำงานภายในบล็อกโค้ด)
- ป้องกันการอ้างอิงแบบวงกลมโดยการติดตามไฟล์ที่ประมวลผลแล้ว
- ไฟล์ที่ไม่มีอยู่จริงจะถูกละเว้นอย่างเงียบๆ
- ความลึกสูงสุดของการรวมไฟล์: 5 ระดับ
- อนุญาตเฉพาะนามสกุลไฟล์ที่เป็นข้อความเท่านั้น (ป้องกันการโหลดภาพ, PDF เป็นต้น)

## การสนับสนุน Frontmatter

ไฟล์ความจำรองรับ YAML frontmatter พร้อมฟิลด์ `paths` สำหรับการฉีดข้อมูลตามเงื่อนไข:

```yaml
---
paths:
  - src/components/**
  - "*.tsx"
---
```

ไฟล์ที่มี `paths` frontmatter จะถูกฉีดเข้าไปเฉพาะเมื่อไฟล์ที่ใช้งานอยู่ตรงกับรูปแบบ glob เท่านั้น

## การกำหนดค่า (Configuration)

- `MAX_MEMORY_CHARACTER_COUNT`: 40,000 ตัวอักษร (สูงสุดที่แนะนำต่อไฟล์)
- การตั้งค่า `claudeMdExcludes`: รูปแบบ Glob เพื่อแยกไฟล์ CLAUDE.md บางไฟล์ออก
- `MAX_INCLUDE_DEPTH`: 5 ระดับของการรวมไฟล์แบบสลับสาย
- ความคิดเห็น HTML ในไฟล์ความจำจะถูกแยกออกก่อนที่การฉีดข้อมูลจะเกิดขึ้น
