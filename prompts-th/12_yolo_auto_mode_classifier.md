# ระบบคัดกรองโหมดอัตโนมัติ (YOLO/Auto-Mode Classifier System)

> **พบใน**: สถาปัตยกรรมภายในของ Claude Code
>
> ระบบการจัดประเภทความปลอดภัย 2 ขั้นตอนที่ตัดสินใจว่าการเรียกใช้เครื่องมือควรได้รับการอนุมัติโดยอัตโนมัติหรือถูกบล็อก นี่คือหัวใจสำคัญของโหมด "อนุมัติอัตโนมัติ (auto-approve)" ของ Claude Code

---

## ภาพรวมสถาปัตยกรรม (Architecture Overview)

ตัวคัดกรอง YOLO เป็น **คอมโพเนนต์ที่มีความสำคัญด้านความปลอดภัยสูงสุด** ของ Claude Code เมื่อผู้ใช้เปิดใช้งาน "โหมดอัตโนมัติ" (เดิมเรียกว่า "โหมด YOLO") ตัวคัดกรองนี้จะประเมินการเรียกใช้เครื่องมือทุกครั้งก่อนที่จะทำงานจริง

### วิธีการทำงาน

1. **พรอมต์พื้นฐาน (Base Prompt)**: โหลดจาก `yolo-classifier-prompts/auto_mode_system_prompt.txt` (รวมอยู่ในตอน build ผ่าน `bun:bundle` feature flag `TRANSCRIPT_CLASSIFIER`)
2. **แม่แบบการอนุญาต (Permissions Template)**: สลับเปลี่ยนในรันไทม์ขึ้นอยู่กับประเภทผู้ใช้:
   - **ผู้ใช้ภายนอก**: `permissions_external.txt`
   - **พนักงาน Anthropic** (`USER_TYPE=ant`): `permissions_anthropic.txt`
3. **กฎของผู้ใช้ (User Rules)**: ส่วนที่ปรับแต่งได้สามส่วนซึ่งนำมาจาก `settings.autoMode`:
   - `allow` — การดำเนินการที่ผู้ใช้อนุญาตอย่างชัดเจน
   - `soft_deny` — การดำเนินการที่ควรถูกบล็อก (แต่ผู้ใช้สามารถแทนที่ได้)
   - `environment` — บริบทเกี่ยวกับสภาพแวดล้อมของผู้ใช้

### โครงสร้างเครื่องมือตัวคัดกรอง (Classifier Tool Schema)

ตัวคัดกรองใช้การเรียกเครื่องมือแบบบังคับ (`classify_result`) สำหรับเอาต์พุตที่มีโครงสร้าง:

```json
{
  "name": "classify_result",
  "description": "รายงานผลการจัดประเภทความปลอดภัยสำหรับการดำเนินการของเอเจนต์",
  "input_schema": {
    "type": "object",
    "properties": {
      "thinking": {
        "type": "string",
        "description": "การใช้เหตุผลทีละขั้นตอนสั้นๆ"
      },
      "shouldBlock": {
        "type": "boolean",
        "description": "ควรบล็อกการดำเนินการ (true) หรืออนุญาต (false)"
      },
      "reason": {
        "type": "string",
        "description": "คำอธิบายสั้นๆ ของการตัดสินใจในการคัดกรอง"
      }
    },
    "required": ["thinking", "shouldBlock", "reason"]
  }
}
```

### การประกอบพรอมต์ระบบ (System Prompt Assembly)

```
buildYoloSystemPrompt(context):
  1. โหลด BASE_PROMPT จาก auto_mode_system_prompt.txt
  2. แทนที่ <permissions_template> ด้วยแม่แบบภายนอก หรือแม่แบบสำหรับ Anthropic
  3. แทนที่ <user_allow_rules_to_replace> ด้วยกฎที่อนุญาตของผู้ใช้
  4. แทนที่ <user_deny_rules_to_replace> ด้วยกฎที่สั่งห้ามของผู้ใช้
  5. แทนที่ <user_environment_to_replace> ด้วยข้อมูลสภาพแวดล้อมของผู้ใช้
```

### การรวมเข้ากับ CLAUDE.md (CLAUDE.md Integration)

ตัวคัดกรองยังได้รับเนื้อหาจากไฟล์ CLAUDE.md ในรูปแบบข้อความนำหน้า (prefixed):

```
ต่อไปนี้คือการกำหนดค่า CLAUDE.md ของผู้ใช้ เหล่านี้คือ
คำแนะนำที่ผู้ใช้จัดเตรียมไว้ให้เอเจนต์และควรได้รับการปฏิบัติ
เสมือนเป็นส่วนหนึ่งของเจตนาของผู้ใช้เมื่อทำการประเมินการดำเนินการ

<user_claude_md>
{เนื้อหา}
</user_claude_md>
```

### เครื่องมือที่ปลอดภัยในรายการอนุญาต (Safe Allowlisted Tools)

เครื่องมือเหล่านี้ได้รับการอนุมัติล่วงหน้าและข้ามขั้นตอนการคัดกรองไปเลย:

- เครื่องมือแบบอ่านอย่างเดียว (การอ่านไฟล์, glob, grep)
- เครื่องมือคัดกรองเอง (`classify_result`)
- ยูทิลิตี้ที่ปลอดภัยต่างๆ

### ลำดับเหตุการณ์บทสนทนา (Conversation Transcript)

ตัวคัดกรองจะได้รับ **ลำดับเหตุการณ์บทสนทนาแบบย่อ**:
- ข้อความจากผู้ใช้: ข้อความอย่างเดียวเท่านั้น
- ข้อความจากผู้ช่วย: **บล็อก tool_use เท่านั้น** (ข้อความจะถูกแยกออกเพื่อป้องกันไม่ให้โมเดลสร้างข้อความที่พยายามโน้มน้าวตัวคัดกรอง)

### คำแนะนำสั่งห้ามสำหรับ PowerShell (PowerShell Deny Guidance)

บน Windows จะมีการเพิ่มกฎสั่งห้ามเพิ่มเติม:
- คำสั่ง PowerShell ที่แก้ไขสถานะระบบ
- การแก้ไข Registry
- คำสั่งจัดการบริการ (Service management)

## การตัดสินใจออกแบบที่สำคัญ

1. **การยกเว้นข้อความ**: บล็อกข้อความของผู้ช่วยจะถูกแยกออกโดยเจตนาจากลำดับเหตุการณ์ เพื่อป้องกันการทำ Prompt Injection ผ่านข้อความที่เขียนโดยโมเดล
2. **เป็นมิตรกับแคช (Cache-friendly)**: ข้อความนำหน้า CLAUDE.md ใช้ `cache_control` เพื่อแคชพรอมต์ข้ามการเรียกตัวคัดกรอง
3. **กำหนดด้วย Feature-gated**: ระบบทั้งหมดอยู่ภายใต้ธง build `TRANSCRIPT_CLASSIFIER`
4. **แม่แบบคู่**: พนักงาน Anthropic จะมีค่าเริ่มต้นที่อนุญาตต่างจากผู้ใช้ภายนอก
5. **การปรับแต่งโดยผู้ใช้**: ระบบสามส่วน (อนุญาต/สั่งห้าม/สภาพแวดล้อม) ช่วยให้ผู้ใช้สามารถปรับแต่งพฤติกรรมของตัวคัดกรองได้
```
