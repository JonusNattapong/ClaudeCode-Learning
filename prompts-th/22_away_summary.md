# สรุปช่วงไม่อยู่ (สรุปเซสชัน) (Away Summary)

> **พบใน**: สถาปัตยกรรมภายในของ Claude Code
> **ฟังก์ชัน:** `buildAwaySummaryPrompt()`
> **โมเดล:** โมเดลขนาดเล็กและเร็ว (Haiku)

---

## วัตถุประสงค์ (Purpose)

สร้างการสรุปย่อ 1-3 ประโยคเมื่อผู้ใช้กลับมาที่เซสชันหลังจากหยุดพักไป แสดงผลในรูปแบบการ์ด "ในขณะที่คุณไม่อยู่ (while you were away)" ใน UI

---

## พรอมต์ระบบ (System Prompt)

พรอมต์ระบบถูกประกอบขึ้นแบบไดนามิกพร้อมคำนำขยายบริบทความจำเลือกใช้งานได้:

```
[ความจำเซสชัน (บริบทที่กว้างขึ้น):
{เนื้อหาความจำ}]

ผู้ใช้เดินออกไปและกำลังกลับมา เขียนข้อความสั้นๆ ไม่เกิน 1-3 ประโยค เริ่มต้นด้วยการระบุงานในระดับภาพรวม — ว่าพวกเขากำลังสร้างหรือแก้ไขจุดบกพร่องอะไร ไม่ใช่รายละเอียดการนำไปใช้งาน จากนั้น: ขั้นตอนถัดไปที่เป็นนามธรรม ข้ามรายงานสถานะและการสรุปการ commit
```

---

## ข้อกำหนดการออกแบบ (Design Constraints)

- **ความกะทัดรัด**: สรุปสั้นๆ 1-3 ประโยคเป๊ะๆ
- **เน้นงานก่อน (Task-first)**: เริ่มต้นด้วยสิ่งที่ผู้ใช้กำลังสร้างหรือแก้ไขจุดบกพร่อง
- **นำไปปฏิบัติได้ (Actionable)**: รวมขั้นตอนถัดไปที่ชัดเจน
- **ไม่มีเสียงรบกวน (No noise)**: ข้ามรายงานสถานะและสรุปการ commit
- **หลีกเลี่ยงรายละเอียดการนำไปใช้งาน**: เน้นที่งานในระดับสูง ไม่ใช่รายละเอียดของโค้ด

## การประมวลผลอินพุต (Input Processing)

- ใช้เฉพาะ 30 ข้อความล่าสุด (`RECENT_MESSAGE_WINDOW = 30`) เพื่อหลีกเลี่ยงข้อผิดพลาด "พรอมต์ยาวเกินไป" ในเซสชันขนาดใหญ่
- รวมเนื้อหาความจำเซสชันสำหรับบริบทที่กว้างขึ้น
- ปิดใช้งานการคิด (thinking) เพื่อให้การตอบสนองรวดเร็ว
- ใช้ `skipCacheWrite: true` เพื่อหลีกเลี่ยงการทำให้แคชปนเปื้อน

## การจัดการข้อผิดพลาด (Error Handling)

- ส่งคืน `null` ในกรณีที่มีการยกเลิก, ลำดับเหตุการณ์ว่างเปล่า หรือเกิดข้อผิดพลาดใดๆ
- ยอมรับข้อยกเว้น `APIUserAbortError` อย่างเงียบเชียบ
