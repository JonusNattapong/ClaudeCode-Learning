# สรุปงานเอเจนต์ (Agent Summary) (ความคืบหน้าเบื้องหลัง)

> **พบใน**: สถาปัตยกรรมภายในของ Claude Code
> **โมเดล:** โมเดลขนาดเล็กและเร็ว (Haiku)

---

## วัตถุประสงค์ (Purpose)

สร้างการอัปเดตความคืบหน้าของงานเบื้องหลังเป็นระยะๆ สำหรับเอเจนต์ย่อย (sub-agents) ที่รันในโหมดผู้ประสานงาน (coordinator mode) เพื่อให้เอเจนต์ตัวแม่รับรู้สถานะการทำงานของแต่ละเอเจนต์ย่อยแบบเรียลไทม์

---

## พรอมต์ (ถอดแบบจากการวิเคราะห์ซอร์สโค้ด) (Prompt)

```
สรุปสิ่งที่เอเจนต์กำลังทำอยู่ตอนนี้ในประโยคสั้นๆ 1 ประโยค ใช้รูปปัจจุบันกาล (present tense)
เน้นไปที่การดำเนินการที่เจาะจง ไม่ใช่ภารกิจในภาพรวม

ดี: "Reading the authentication middleware to understand token validation" (กำลังอ่าน middleware การตรวจสอบสิทธิ์เพื่อทำความเข้าใจการตรวจสอบโทเคน)
ดี: "Running pytest on the user service after fixing the import error" (กำลังรัน pytest ในส่วนการรันผู้ใช้หลังจากแก้ไขข้อผิดพลาดการนำเข้า)
ไม่ดี: "Working on the task" (กว้างเกินไป)
ไม่ดี: "The agent is currently in the process of examining..." (เยิ่นเย้อเกินไป)
```

---

## ข้อกำหนดการออกแบบ (Design Constraints)

- **ประโยคเดียว**: สูงสุดหนึ่งประโยค, ใช้รูปปัจจุบันกาล (present tense)
- **เจาะจงการดำเนินการ**: อธิบายการดำเนินการปัจจุบัน ไม่ใช่เป้าหมายในภาพรวม
- **ไม่มีการบรรยายเสริม**: อย่าบรรยายตัวเอเจนต์เอง แต่ให้อธิบายสิ่งที่มันกำลังทำ
- **รูปปัจจุบันกาล (Present tense)**: ใช้คำกริยาที่แสดงความต่อเนื่องเสมอ ("Reading", "Running", "Fixing" - กำลังอ่าน, กำลังรัน, กำลังแก้ไข)

## จุดเชื่อมต่อการรวม (Integration Points)

- รันด้วยตัวจับเวลาเป็นระยะๆ ระหว่างโหมดผู้ประสานงาน (coordinator mode)
- ผลลัพธ์จะถูกแสดงในหน้าต่างสถานะของผู้ประสานงาน
- ช่วยให้เอเจนต์หลักตัดสินใจได้ว่าเมื่อใดควรเข้าไปตรวจสอบการทำงานของเอเจนต์ย่อย
- จะทำงานเฉพาะเมื่อเอเจนต์ย่อยกำลังเรียกใช้เครื่องมืออยู่จริงๆ เท่านั้น
