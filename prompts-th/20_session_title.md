# ตัวสร้างชื่อเซสชัน (Session Title Generator)

> **พบใน**: สถาปัตยกรรมภายในของ Claude Code
> **ตัวแปร:** `SESSION_TITLE_PROMPT`
> **โมเดล:** Haiku (ผ่าน `queryHaiku`)
> **รูปแบบเอาต์พุต:** JSON (`{ "title": "..." }`)

---

## วัตถุประสงค์ (Purpose)

สร้างชื่อเซสชันที่กระชับและใช้รูปแบบตัวพิมพ์ใหญ่เฉพาะคำแรก (sentence-case) (3-7 คำ) จากเนื้อหาการสนทนา ทำหน้าที่เป็นแหล่งข้อมูลเดียว (single source of truth) สำหรับชื่อเซสชันที่สร้างโดย AI ทั่วทุกหน้าจอ (SDK, เซสชันระยะไกล CCR, REPL bridge)

---

## พรอมต์ระบบ (System Prompt)

```
สร้างชื่อเซสชันที่กระชับและใช้รูปแบบตัวพิมพ์ใหญ่เฉพาะคำแรก (sentence-case) (3-7 คำ) ที่ครอบคลุมหัวข้อหลักหรือเป้าหมายของเซสชันการเขียนโค้ดนี้ ชื่อควรชัดเจนเพียงพอที่ผู้ใช้จะจำเซสชันได้จากรายการ ใช้รูปแบบ sentence case: เขียนตัวพิมพ์ใหญ่เฉพาะคำแรกและคำเฉพาะเจาะจงเท่านั้น

ส่งคืนเป็น JSON พร้อมฟิลด์ "title" เพียงฟิลด์เดียว

ตัวอย่างที่ดี:
{"title": "Fix login button on mobile"} (แก้ปุ่มเข้าสู่ระบบบนมือถือ)
{"title": "Add OAuth authentication"} (เพิ่มการยืนยันตัวตน OAuth)
{"title": "Debug failing CI tests"} (แก้จุดพกพร่องการทดสอบ CI ที่ล้มเหลว)
{"title": "Refactor API client error handling"} (ปรับปรุงการจัดการข้อผิดพลาด API client)

ตัวอย่างที่ไม่ดี (กว้างเกินไป): {"title": "Code changes"} (การเปลี่ยนโค้ด)
ตัวอย่างที่ไม่ดี (ยาวเกินไป): {"title": "Investigate and fix the issue where the login button does not respond on mobile devices"} (ตรวจสอบและแก้ไขปัญหาที่ปุ่มเข้าสู่ระบบไม่ตอบสนองบนอุปกรณ์มือถือ)
ตัวอย่างที่ไม่ดี (ตัวพิมพ์ใหญ่ผิด): {"title": "Fix Login Button On Mobile"}
```

## การประมวลผลอินพุต (Input Processing)

ฟังก์ชัน `extractConversationText()` จะแผ่อาร์เรย์ข้อความให้เป็นสตริงข้อความเดียว:
- ข้ามข้อความเมตาและข้อความที่ไม่ใช่ของมนุษย์
- ตัดทอนเอาเฉพาะข้อความ 1,000 ตัวอักษรสุดท้ายเพื่อให้บริบทล่าสุดมีความสำคัญ
- กรองโดย `msg.origin.kind === 'human'`

## จุดเชื่อมต่อการรวม (Integration Points)

- **เส้นทางพิมพ์ SDK (SDK print path)**: การสร้างชื่อเซสชันแบบไม่โต้ตอบ
- **เซสชันระยะไกล CCR (CCR remote sessions)**: ชื่อเซสชันแบบโต้ตอบผ่าน `useRemoteSession`
- **REPL bridge**: อัปเกรดชื่อเซสชันโดยไม่ต้องรอให้เสร็จสิ้น (fire-and-forget) หลังจากมีข้อความจากผู้ใช้ 3 ข้อความ
