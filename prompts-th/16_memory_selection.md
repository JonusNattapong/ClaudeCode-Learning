# พรอมต์การเลือกความจำ (Memory Selection Prompt)

> **พบใน**: สถาปัตยกรรมภายในของ Claude Code
>
> เลือกไฟล์ความจำสูงสุด 5 ไฟล์จากไดเรกทอรีความจำ `.claude/` ของผู้ใช้ที่เกี่ยวข้องกับคำค้นหาปัจจุบัน ใช้ Sonnet เพื่อการจับคู่เชิงความหมายที่มีคุณภาพสูง

---

## พรอมต์ระบบ (System Prompt)

```
คุณกำลังเลือกความจำ (memories) ที่จะมีประโยชน์ต่อ Claude Code เมื่อประมวลผลคำค้นหาของผู้ใช้ คุณจะได้รับคำค้นหาของผู้ใช้และรายการไฟล์ความจำที่มีชื่อไฟล์และคำอธิบาย

ส่งคืนรายการชื่อไฟล์สำหรับความจำที่จะเป็นประโยชน์อย่างชัดเจนต่อ Claude Code ในขณะที่ประมวลผลคำค้นหาของผู้ใช้ (สูงสุด 5 รายการ) รวมเฉพาะความจำที่คุณมั่นใจว่าจะช่วยได้ตามชื่อและคำอธิบายของพวกมันเท่านั้น
- หากคุณไม่แน่ใจว่าความจำจะมีประโยชน์ในการประมวลผลคำค้นหาของผู้ใช้หรือไม่ ก็ไม่ต้องรวมไว้ในรายการของคุณ ให้เลือกเฟ้นอย่างรอบคอบ
- หากไม่มีความจำในรายการที่ดูแล้วจะมีประโยชน์อย่างชัดเจน คุณสามารถส่งคืนรายการว่างได้
- หากมีการแสดงรายการเครื่องมือที่เพิ่งใช้ล่าสุด ห้ามเลือกความจำที่เป็นการอ้างอิงการใช้งานหรือเอกสาร API สำหรับเครื่องมือเหล่านั้น (Claude Code กำลังใช้งานพวกมันอยู่แล้ว) แต่คุณยังควรเลือกความจำที่มีคำเตือน, กลเม็ด (gotchas) หรือปัญหาที่ทราบเกี่ยวกับเครื่องมือเหล่านั้น — การใช้งานจริงคือช่วงเวลาที่สิ่งเหล่านั้นสำคัญที่สุด
```

## แม่แบบพรอมต์ผู้ใช้ (User Prompt Template)

```
คำค้นหา (Query): {user's query}

ความจำที่มีอยู่ (Available memories):
{รายการไฟล์ความจำพร้อมชื่อไฟล์และคำอธิบาย}

เครื่องมือที่ใช้ล่าสุด (Recently used tools): {เครื่องมือ 1, เครื่องมือ 2, ...}
```

## โครงสร้างเอาต์พุต (Structured JSON Schema)

```json
{
  "type": "object",
  "properties": {
    "selected_memories": {
      "type": "array",
      "items": { "type": "string" }
    }
  },
  "required": ["selected_memories"],
  "additionalProperties": false
}
```

## รายละเอียดทางเทคนิค (Technical Details)

| คุณสมบัติ | ค่า |
|----------|-------|
| **โมเดล** | Sonnet |
| **แหล่งที่มาของ Query** | `memdir_relevance` |
| **จำนวนโทเคนสูงสุด** | 256 |
| **รูปแบบเอาต์พุต** | JSON ตามโครงสร้างที่กำหนด (Structured JSON schema) |
| **ผลลัพธ์สูงสุด** | 5 ไฟล์ความจำ |
| **ข้อยกเว้น** | `MEMORY.md` (ถูกใส่ไว้ในพรอมต์ระบบแล้ว) และไฟล์ที่เคยแสดงไปก่อนหน้าแล้ว |
| **การกรองเครื่องมือ** | ข้ามเอกสาร API สำหรับเครื่องมือที่เพิ่งใช้ แต่เก็บกลเม็ด/คำเตือนไว้ |

## ขั้นตอนการเลือก (Selection Pipeline)

```
คำค้นหาจากผู้ใช้มาถึง
    │
    ├── สแกนไดเรกทอรีความจำเพื่อหาไฟล์ .md ที่มี frontmatter
    │   └── ดึงชื่อไฟล์ + คำอธิบายจากหัวข้อ YAML
    │
    ├── กรองความจำที่เคยแสดงไปแล้วออก (จากรอบก่อนหน้า)
    │
    ├── จัดรูปแบบรายการ (manifest): "ชื่อไฟล์.md — คำอธิบาย"
    │
    ├── รวมรายการเครื่องมือที่ใช้ล่าสุด (เพื่อหลีกเลี่ยงเอกสาร API ที่ซ้ำซ้อน)
    │
    └── Sonnet เลือกชื่อไฟล์สูงสุด 5 รายการ → {"selected_memories": ["file1.md", ...]}
        └── ตรวจสอบความถูกต้องกับชื่อไฟล์จริงบนดิสก์
```
