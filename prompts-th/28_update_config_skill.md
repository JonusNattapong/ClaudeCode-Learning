# สกิลการอัปเดตการกำหนดค่า (Update Config Skill) (/update-config)

> **พบใน**: สถาปัตยกรรมภายในของ Claude Code
> **การลงทะเบียน:** `registerBundledSkill('update-config', ...)`
> **ความพร้อมใช้งาน:** ผู้ใช้ทุกคน

---

## วัตถุประสงค์ (Purpose)

จัดการไฟล์การกำหนดค่า `settings.json` รวมถึง hooks, อาร์เรย์การอนุญาต และการตั้งค่าทั่วไป ให้ส่วนติดต่อแนะนำ (guided interface) สำหรับการแก้ไขลำดับชั้นการกำหนดค่าของ Claude Code

---

## พรอมต์ (ถอดแบบจากการวิเคราะห์ไบนารี) (Prompt)

พรอมต์มีความยาวมาก (~476 บรรทัด) และครอบคลุมเนื้อหาหลักสามส่วน:

### ลำดับชั้นของไฟล์การตั้งค่า (Settings File Hierarchy)

```
การตั้งค่าจะถูกโหลดจากสามระดับ (ความสำคัญสูงสุดไปหาต่ำสุด):
1. การตั้งค่าโปรเจกต์: .claude/settings.json (ถูกตรวจสอบเข้าสู่ repo)
2. การตั้งค่าผู้ใช้: ~/.claude/settings.json (ส่วนตัว, ใช้ทุกโปรเจกต์)
3. การตั้งค่าองค์กร: /etc/claude-code/settings.json (จัดการโดยแอดมิน)

การตั้งค่าที่มีความสำคัญสูงกว่าจะถูกนำไปแทนที่การตั้งค่าที่มีความสำคัญต่ำกว่า
```

### ระบบฮุค (Hook System)

สกิลรวบรวมข้อมูลเกี่ยวกับวงจรชีวิตของฮุคไว้อย่างครบถ้วน:

```
ฮุค (Hooks) คือคำสั่งที่รัน ณ จุดเฉพาะเจาะจงในวงจรชีวิตของ Claude Code:
- PreToolUse: รันก่อนที่จะมีการใช้งานเครื่องมือ สามารถบล็อกเครื่องมือได้
- PostToolUse: รันหลังจากใช้เครื่องมือเสร็จสิ้น สามารถแก้ไขผลลัพธ์ได้
- PreCompact: รันก่อนการบีบอัดบทสนทนา
- PostCompact: รันหลังการบีบอัดบทสนทนา
- Notification: รันเมื่อ Claude ต้องการแจ้งเตือนผู้ใช้
- Stop: รันเมื่อเวลาของ Claude สิ้นสุดลง (turn ends)

แต่ละฮุคจะได้รับข้อมูล JSON ทาง stdin พร้อมระบุช่องข้อมูลดังนี้:
- session_id: ID เซสชันปัจจุบัน
- tool_name: ชื่อเครื่องมือ (PreToolUse/PostToolUse)
- tool_input: พารามิเตอร์อินพุตของเครื่องมือ (PreToolUse/PostToolUse)
- tool_output: เอาต์พุตของเครื่องมือ (PostToolUse เท่านั้น)

รหัสออกจากฮุค (Hook exit codes):
- 0: สำเร็จ, ทำงานต่อตามปกติ
- 2: บล็อกเครื่องมือ (PreToolUse เท่านั้น)
- อื่นๆ: บันทึกข้อผิดพลาดแต่ให้ทำงานต่อไป
```

### อาร์เรย์การอนุญาต (Permission Arrays)

```
การตั้งค่าการอนุญาตควบคุมว่าเครื่องมือใดสามารถทำงานได้โดยไม่ต้องมีการยืนยันจากผู้ใช้:
- allowedTools: ชื่อเครื่องมือหรือรูปแบบที่อนุญาตแบบอัตโนมัติ
- deniedTools: ชื่อเครื่องมือหรือรูปแบบที่ถูกบล็อกเสมอ
- trust: ระดับความน่าเชื่อถือสำหรับหมวดหมู่เครื่องมือต่างๆ
```

## หมายเหตุเชิงสถาปัตยกรรม

- สกิลแยกความแตกต่างระหว่างเครื่องมือคอนฟิกแบบ "ง่าย" (การตั้งค่า key-value) และการดำเนินการแบบ "ซับซ้อน" (ฮุค, การอนุญาต)
- สำหรับการแก้ไขฮุค จะใช้เครื่องมือ Edit โดยตรงกับไฟล์ settings.json
- ตรวจสอบความถูกต้องของไวยากรณ์ JSON หลังจากแก้ไขแต่ละครั้งเสมอ
- รองรับการตั้งค่าทั้งสามระดับในส่วนติดต่อเดียว
