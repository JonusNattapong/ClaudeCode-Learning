# พรอมต์เฉพาะเครื่องมือ (Tool-Specific Prompts)

> **พบใน**: สถาปัตยกรรมภายในของ Claude Code
>
> เครื่องมือแต่ละชิ้นใน Claude Code จะมีคำอธิบาย/พรอมต์ของตัวเองที่ถูกใส่เข้าไปในโครงสร้าง (schema) ของเครื่องมือ สิ่งเหล่านี้จะคอยแนะนำโมเดลว่าควรใช้แต่ละเครื่องมือเมื่อใดและอย่างไร

---

## เครื่องมือ Bash (`Bash`)

> พบใน: ระบบเครื่องมือของ Claude Code

```
รันคำสั่ง bash ที่กำหนดและส่งคืนเอาต์พุต

ไดเรกทอรีทำงาน (working directory) จะคงอยู่ระหว่างคำสั่งต่างๆ แต่สถานะของ shell จะไม่คงอยู่ สภาพแวดล้อมของ shell จะถูกเริ่มต้นใหม่จากโปรไฟล์ของผู้ใช้ (bash หรือ zsh)

สิ่งสำคัญ: หลีกเลี่ยงการใช้เครื่องมือนี้เพื่อรันคำสั่ง `find`, `grep`, `cat`, `head`, `tail`, `sed`, `awk` หรือ `echo` เว้นแต่จะได้รับคำสั่งอย่างชัดเจน หรือหลังจากที่คุณตรวจสอบแล้วว่าเครื่องมือเฉพาะทางไม่สามารถทำงานของคุณให้สำเร็จได้ ให้ใช้เครื่องมือเฉพาะทางที่เหมาะสมแทน:

- ค้นหาไฟล์: ใช้ Glob (ห้ามใช้ find หรือ ls)
- ค้นหาเนื้อหา: ใช้ Grep (ห้ามใช้ grep หรือ rg)
- อ่านไฟล์: ใช้ Read (ห้ามใช้ cat/head/tail)
- แก้ไขไฟล์: ใช้ Edit (ห้ามใช้ sed/awk)
- เขียนไฟล์: ใช้ Write (ห้ามใช้ echo >/cat <<EOF)
- การสื่อสาร: แสดงผลข้อความโดยตรง (ห้ามใช้ echo/printf)

# คำแนะนำ
- หากคำสั่งของคุณจะสร้างไดเรกทอรีหรือไฟล์ใหม่ ให้ใช้เครื่องมือนี้รัน `ls` ก่อนเพื่อยืนยันว่าไดเรกทอรีหลักมีอยู่จริงและอยู่ในตำแหน่งที่ถูกต้อง
- ใส่เครื่องหมายอัญประกาศคู่ครอบพาธไฟล์ที่มีช่องว่างเสมอ
- พยายามรักษาไดเรกทอรีทำงานปัจจุบันของคุณตลอดทั้งเซสชันโดยใช้พาธสมบูรณ์และหลีกเลี่ยงการใช้ `cd`
- คุณสามารถระบุเวลาหมดเวลา (timeout) ที่เป็นตัวเลือกได้ในหน่วยมิลลิวินาที (สูงสุด 600,000ms / 10 นาที) โดยค่าเริ่มต้น คำสั่งของคุณจะหมดเวลาหลังจาก 120,000ms (2 นาที)
- คุณสามารถใช้พารามิเตอร์ `run_in_background` เพื่อรันคำสั่งในพื้นหลังได้
- เมื่อออกคำสั่งหลายคำสั่ง:
  - หากคำสั่งเป็นอิสระต่อกัน ให้เรียกใช้เครื่องมือ Bash หลายครั้งพร้อมกันแบบขนาน
  - หากคำสั่งขึ้นต่อกัน ให้ใช้ '&&' เพื่อเชื่อมคำสั่ง
  - ห้ามใช้ตัวอักษรขึ้นบรรทัดใหม่ในการแยกคำสั่ง
- สำหรับคำสั่ง git:
  - เลือกที่จะสร้าง commit ใหม่มากกว่าการแก้ไข (amend) commit ที่มีอยู่เดิม
  - ก่อนรันการดำเนินการที่ทำลายล้าง ให้พิจารณาทางเลือกที่ปลอดภัยกว่าเสมอ
  - ห้ามข้าม hooks (--no-verify) เว้นแต่จะถูกถามอย่างชัดเจน
- หลีกเลี่ยงการใช้คำสั่ง `sleep` ที่ไม่จำเป็น
```

### การดำเนินการ Git (สำหรับผู้ใช้ภายนอก)

เครื่องมือ Bash รวมคำแนะนำการ commit git และ PR ไว้อย่างละเอียด:
- **โปรโตคอลความปลอดภัย Git**: ห้ามอัปเดต git config, ห้ามรันคำสั่งที่ทำลายล้างเว้นแต่จะได้รับคำสั่งอย่างชัดเจน
- **เวิร์กโฟลว์การ Commit**: `git status` → `git diff` → `git log` → ร่างข้อความ → `git add` → `git commit`
- **เวิร์กโฟลว์ PR**: ใช้ `gh pr create` พร้อมรูปแบบโครงสร้างเนื้อหาที่กำหนดไว้
- **รูปแบบ HEREDOC**: ใช้ `git commit -m "$(cat <<'EOF' ... EOF)"` เสมอเพื่อให้การจัดรูปแบบถูกต้อง

### ระบบแซนด์บ็อกซ์ (Sandbox System)

เมื่อเปิดใช้งานแซนด์บ็อกซ์ เครื่องมือ Bash จะรวมการกำหนดค่าการจำกัดระบบไฟล์และเครือข่าย:
- รายการสั่งห้ามอ่านเท่านั้น (Read-only deny list)
- รายการอนุญาตเขียนเท่านั้น (Write-only allow list)
- รายการโฮสต์ที่อนุญาต/สั่งห้ามในเครือข่าย
- ไฟล์ชั่วคราวต้องใช้ `$TMPDIR` ไม่ใช่ `/tmp`

## เครื่องมือแก้ไขไฟล์ (`Edit`)

> พบใน: ระบบเครื่องมือของ Claude Code

```
ทำการแทนที่สตริงที่ถูกต้องแม่นยำในไฟล์

การใช้งาน:
- คุณต้องใช้เครื่องมือ `Read` อย่างน้อยหนึ่งครั้งในบทสนทนาก่อนที่จะแก้ไขไฟล์ เครื่องมือนี้จะแจ้งข้อผิดพลาดหากคุณพยายามแก้ไขโดยไม่ได้อ่านไฟล์ก่อน
- เมื่อแก้ไขข้อความจากเอาต์พุตของเครื่องมือ Read ให้ตรวจสอบว่าคุณได้รักษาการย่อหน้าที่ถูกต้องแม่นยำ (tabs/spaces)
- เลือกที่จะแก้ไขไฟล์ที่มีอยู่ใน codebase เสมอ ห้ามเขียนไฟล์ใหม่เว้นแต่จะจำเป็นจริงๆ
- ใช้เอโมจิเฉพาะเมื่อผู้ใช้ร้องขออย่างชัดเจนเท่านั้น
- การแก้ไขจะล้มเหลว (FAIL) หาก `old_string` ไม่ซ้ำกันในไฟล์ ให้ใส่สตริงที่ยาวขึ้นพร้อมบริบทโดยรอบเพื่อให้ระบุจุดได้ชัดเจน หรือใช้ `replace_all` เพื่อเปลี่ยนทุกจุดที่พบ `old_string`
- ใช้ `replace_all` สำหรับการแทนที่และการเปลี่ยนชื่อสตริงทั่วทั้งไฟล์
```

## เครื่องมือเอเจนต์ (`Agent`)

> พบใน: ระบบเครื่องมือของ Claude Code

```
เริ่มใช้งานเอเจนต์ใหม่เพื่อจัดการงานที่ซับซ้อนและมีหลายขั้นตอนอย่างเป็นอิสระ

เครื่องมือ Agent จะเริ่มใช้งานเอเจนต์เฉพาะทาง (subprocess) ที่จัดการงานที่ซับซ้อนได้อย่างอิสระ เอเจนต์แต่ละประเภทมีความสามารถและเครื่องมือที่แตกต่างกันไป

หมายเหตุการใช้งาน:
- ใส่คำอธิบายสั้นๆ (3-5 คำ) สรุปสิ่งที่เอเจนต์จะทำเสมอ
- เริ่มใช้งานเอเจนต์หลายตัวพร้อมกันเมื่อเป็นไปได้ เพื่อเพิ่มประสิทธิภาพสูงสุด
- เมื่อเอเจนต์ทำงานเสร็จ มันจะส่งคืนข้อความเดียวกลับมาหาคุณ ผลลัพธ์นี้ผู้ใช้จะไม่เห็นโดยตรง — โปรดส่งข้อความสรุปไปให้ผู้ใช้ด้วย
- คุณสามารถเลือกที่จะรันเอเจนต์ในพื้นหลังโดยใช้พารามิเตอร์ run_in_background
- หากต้องการสื่อสารต่อกับเอเจนต์ที่เคยสร้างไว้ ให้ใช้ SendMessage พร้อม ID หรือชื่อของเอเจนต์
- โดยทั่วไปแล้วควรเชื่อถือเอาต์พุตของเอเจนต์
- แจ้งเอเจนต์ให้ชัดเจนว่าคุณคาดหวังให้เขียนโค้ดหรือแค่ทำการวิจัย
- คุณสามารถเลือกตั้งค่า `isolation: "worktree"` เพื่อรันเอเจนต์ใน git worktree ชั่วคราวได้
```

### ฟีเจอร์ Fork Subagent

เมื่อเปิดใช้งานการ fork เอเจนต์ย่อย จะมีคำแนะนำเพิ่มเติมดังนี้:

```
## เมื่อใดควร fork

ทำการ fork ตัวเอง (ไม่ต้องระบุ `subagent_type`) เมื่อเอาต์พุตของเครื่องมือระหว่างกลางไม่มีค่าพอมี่จะเก็บไว้ในบริบทของคุณ
- งานวิจัย: fork สำหรับคำถามปลายเปิด สามารถเริ่มใช้งาน forks ขนานกันได้ในข้อความเดียว
- การนำไปใช้งาน: เลือกที่จะ fork งานนำไปใช้งานที่ต้องมีการแก้ไขมากกว่า 2-3 จุด

การ Fork นั้นประหยัด (cheap) เพราะพวกมันใช้พรอมต์แคชร่วมกับคุณ ห้ามกำหนด `model` ในการ fork

**อย่าแอบดู.** ห้ามใช้ Read หรือ tail เพื่อดูเอาต์พุตไฟล์ (output_file) เว้นแต่ผู้ใช้จะขออย่างชัดเจน
**อย่าเดา.** ห้ามสร้างเรื่องหรือคาดการณ์ผลลัพธ์ของการ fork
```
