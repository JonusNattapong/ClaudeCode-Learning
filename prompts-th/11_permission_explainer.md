# พรอมต์ระบบอธิบายการอนุญาต (Permission Explainer System Prompt)

> **พบใน**: สถาปัตยกรรมภายในของ Claude Code
>
> การสอบถามย่อย (side-query) น้ำหนักเบาที่อธิบายว่าเครื่องมือ/คำสั่งนั้นทำอะไร ทำไมจึงต้องรัน และระดับความเสี่ยงก่อนที่ผู้ใช้จะอนุมัติ ใช้เอาต์พุตแบบมีโครงสร้างเพื่อให้แน่ใจว่าจะได้รับ JSON กลับมา

---

## พรอมต์ระบบ (System Prompt)

```
วิเคราะห์คำสั่งเชลล์และอธิบายว่าคำสั่งเหล่านั้นทำอะไร ทำไมคุณถึงรันคำสั่งเหล่านั้น และความเสี่ยงที่อาจเกิดขึ้น
```

## โครงสร้างเอาต์พุต (Structured Output Schema) (เครื่องมือ: `explain_command`)

```json
{
  "explanation": "คำสั่งนี้ทำอะไร (1-2 ประโยค)",
  "reasoning": "ทำไมคุณถึงรันคำสั่งนี้ เริ่มต้นด้วย 'ฉัน' - เช่น 'ฉันจำเป็นต้องตรวจสอบเนื้อหาไฟล์'",
  "risk": "สิ่งที่อาจผิดพลาดได้ ไม่เกิน 15 คำ",
  "riskLevel": "LOW | MEDIUM | HIGH"
}
```

### คำจำกัดความระดับความเสี่ยง (Risk Level Definitions)

| ระดับ | คำอธิบาย | ตัวอย่าง |
|-------|-------------|---------|
| `LOW` | เวิร์กโฟลว์การพัฒนาที่ปลอดภัย | `ls`, `cat`, `git status` |
| `MEDIUM` | การเปลี่ยนแปลงที่กู้คืนได้ | การแก้ไขไฟล์, `npm install` |
| `HIGH` | อันตราย/ย้อนกลับไม่ได้ | `rm -rf`, `DROP TABLE`, force push |

## พฤติกรรมขณะรันไทม์ (Runtime Behavior)

- ใช้ **โมเดลลูปหลัก** (ไม่ใช่โมเดล Haiku ที่แยกต่างหาก)
- ทำงานเป็น **การสอบถามย่อย (side query)** ควบคู่ไปกับหน้าต่างขออนุญาต
- ดึงข้อความจากผู้ช่วยล่าสุด 3 ข้อความ (ไม่เกิน 1,000 ตัวอักษร) สำหรับบริบท "ทำไมถึงรัน"
- สามารถปิดใช้งานได้โดยการกำหนดค่าของผู้ใช้: `permissionExplainerEnabled: false`
```
